# Working Rules

## Maintenance

- Keep `.codex` rules, context, and memory aligned with meaningful project changes.
- When features, workflows, or team expectations change, update the relevant `.codex` files in the same stream of work.
- Prefer updating the smallest relevant file instead of letting Codex-side docs drift.
- Treat the user shorthand `sync` as: update `.codex` context/rules and the repo's Claude/rules docs to match the current project state.
