Update `.codex` rules files when project behavior, features, or workflow expectations change.

**Why:** The user wants Codex-side agents, memory, and context to stay current with the real project instead of becoming a stale one-time migration.

**How to apply:** When a change materially affects features, architecture, workflow, or review guidance, update the relevant file under `.codex` as part of that work.
